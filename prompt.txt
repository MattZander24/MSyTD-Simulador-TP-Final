Desarrollá una aplicación de escritorio completa en Python para resolver mediante simulación el problema "Juego EBRIO" de la materia Modelos, Simulación y Teoría de la Decisión.

# Objetivo

Implementar un simulador gráfico que permita analizar una caminata aleatoria bidimensional (Random Walk) de un borracho que parte desde el origen (0,0) y en cada esquina decide aleatoriamente moverse una calle hacia:

* Norte
* Sur
* Este
* Oeste

Todas las direcciones deben tener probabilidad uniforme:

P = 0.25

El simulador debe permitir realizar simulaciones individuales y simulaciones Monte Carlo masivas para estimar probabilidades.

# Tecnologías obligatorias

* Python 3.12+
* CustomTkinter para la interfaz gráfica
* Matplotlib para gráficos
* NumPy cuando sea conveniente
* Arquitectura modular

# Modelo del problema

El borracho comienza siempre en:

(0,0)

Cada paso equivale a una calle.

Las coordenadas deben manejarse como:

(x,y)

Movimientos:

Norte  -> (x, y+1)
Sur    -> (x, y-1)
Este   -> (x+1, y)
Oeste  -> (x-1, y)

# Distancia utilizada

Usar exclusivamente distancia Manhattan.

Definición:

distancia = |x| + |y|

# Definición de encontrar las llaves

Las llaves estarán ubicadas en una coordenada configurable.

El borracho encuentra las llaves si en cualquier momento del recorrido pasa por dicha coordenada.

No es necesario terminar sobre las llaves.

Ejemplo:

Si las llaves están en (2,1) y el recorrido visita (2,1) en el paso 4, entonces:

encontro_llaves = True

aunque termine en otra posición.
